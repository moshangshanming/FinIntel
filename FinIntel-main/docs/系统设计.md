# 系统设计

## 1. 系统技术架构

系统采用前后端分离架构。后端负责新闻采集、清洗、去重、分析、入库和接口服务；前端负责可视化展示；数据库负责存储新闻、事件、分析结果和知识图谱数据。

## 2. 系统流程

```text
模拟新闻数据
→ 新闻采集
→ 新闻清洗
→ 去重融合
→ 智能分析
→ SQLite 入库
→ 知识图谱构建
→ 前端可视化展示
```

## 3. 包设计

后端主要分为 models、crawlers、services、repositories、api、utils 六个包：

- models：定义数据对象；
- crawlers：实现新闻采集；
- services：实现核心业务逻辑；
- repositories：实现数据库访问；
- api：实现 HTTP 接口；
- utils：实现通用工具。
